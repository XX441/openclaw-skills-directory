# SSOT Connection Layer

**Created:** 2026-05-19  
**Purpose:** Define how Java connects ideas, documents, projects, and strategic dots across the Drive SSOT without confusing the method with Obsidian.

---

## 1. Name

Use: **SSOT Connection Layer**.

Optional technical name: **SSOT Knowledge Graph**.

Avoid saying “Obsidian” as the feature name. Obsidian is only one possible interface/tool. The actual capability is the connection layer that sits on top of Google Drive SSOT.

---

## 2. What It Does

The SSOT Connection Layer turns isolated files into an operating knowledge system by maintaining:

1. **Canonical source files** — the actual Drive SSOT artifacts.
2. **Project status files** — current state, owner, stage, next action.
3. **Action trackers** — living execution lists with triggers and owners.
4. **Connection maps / briefs** — derived notes explaining what a new document connects to.
5. **Content index** — searchable map of important files and clusters.
6. **Memory pointers** — compact durable reminders in `MEMORY.md`, `HEARTBEAT.md`, and daily logs.

---

## 3. When To Create A Connection Brief

Create a connection brief when a new file is strategically meaningful and touches more than one project, partner, BD lane, standard, event, or operating system.

Examples:
- WUWM Tianjin Conference PDF
- GACC Order 280 policy brief
- Ming Pao media coverage
- Vietnam Trust Corridor strategy
- Awwws WorldPass commercial logic
- Partner profiles that unlock other relationships

---

## 4. Standard Output Pattern

For important new source material:

1. **Normalize / place the source file** in the correct Drive folder.
2. **Create or update a derived brief** with:
   - source facts
   - strategic interpretation
   - connected SSOT nodes
   - action implications
   - evidence / claim warnings
3. **Update status / tracker files** if next actions changed.
4. **Update `memory/ssot-content-index.md`** with the cluster and cross-file links.
5. **Update `HEARTBEAT.md`** if it changes active tracking.
6. **Promote to `MEMORY.md`** only if it changes durable strategy or operating rules.
7. **Log to daily note**.

---

## 5. Rule

Do not over-link everything. Only connect what affects decisions, execution, credibility, or future retrieval.

Good connection layer = fewer, stronger links.
Bad connection layer = Obsidian-style link clutter with no operational value.
