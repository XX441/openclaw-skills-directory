# Pre-Share Checklist

**Mandatory: Run every item before sending any document externally** (email, WhatsApp, shared link, partner review).

## 1. Version Consistency
- [ ] Filename version matches content version (see [version-rules.md](version-rules.md))
- [ ] No stale version numbers in document body

## 2. Archive Clean
- [ ] Old versions moved to `_Archive/` (not deleted, not still in working folder)
- [ ] Only ONE active version exists in the working folder

## 3. File Integrity
- [ ] File opens and renders correctly in target application
- [ ] All tables, images, links work as expected
- [ ] PDF output (if applicable) looks correct at A4

## 4. Correct Lane Folder
- [ ] File is in the correct Google Drive lane (01/02/03/04)
- [ ] Folder path makes sense for external recipients who may browse

## 5. No Placeholders or Internal Notes
- [ ] No `TBC` / `待確認` / `TODO` markers visible
- [ ] No internal tone notes, moderator comments, or draft annotations
- [ ] No "[INTERNAL]" or "[DRAFT]" watermarks/badges

## 6. Metadata & Branding
- [ ] Logo and branding correct for the intended audience
- [ ] Document title, date, and author fields populated
- [ ] Change log reflects the version being shared

## 7. Recipient-Ready
- [ ] Language and tone appropriate for the external audience
- [ ] Confidentiality markings correct (if any)
- [ ] Contact info and next steps clear

## Automation Note

`ssot-upload` handles items 1 and 2 automatically at upload time. Items 3–7 must be checked manually before sending.
